## 1. 项目概述

### 1.1 产品定位
一款轻量级的每日用药提醒小程序，帮助用户养成按时服药的习惯。

### 1.2 核心价值
- **简单**：无需注册，打开即用
- **提醒**：定时推送，不再忘服药
- **记录**：记录服药历史，掌握健康数据

### 1.3 目标用户
- 需要长期服药的慢性病患者
- 服用保健品/维生素的亚健康人群
- 记忆力减退的老年人（可子女协助设置）

## 4. 详细功能描述

### 4.1 每日打卡 (F01)

**交互流程：**
1. 用户打开页面
2. 看到"你今天吃药了吗？"大字
3. 点击"是的，吃过了" → 记录今日已服药
4. 点击"还没有" → 无操作，提示"记得稍后服用"

**状态：**
- 今日已打卡：显示 ✅ 已完成
- 今日未打卡：显示 ❌ 待完成

### 4.2 药物管理 (F02)

**添加药物：**
- 药物名称（必填）
- 提醒时间（必填，默认 09:00）
- 剂量说明（选填，如"1片"、"2粒"）

**编辑/删除：**
- 支持编辑已添加的药物
- 支持删除药物

### 4.3 服药提醒 (F04)

**触发条件：**
- 到达预设提醒时间

**提醒方式：**
- 浏览器桌面通知（Notification API）
- 标题：`💊 服药提醒`
- 内容：`该服用 [药物名称] ([剂量]) 了！`

**权限处理：**
- 首次使用时请求通知权限
- 用户拒绝时显示降级提示

### 4.4 本地存储 (F03)

**存储结构：**
```json
{
  "medicines": [
    { "name": "维生素C", "time": "09:00", "dose": "1片" }
  ],
  "records": {
    "2026-03-17": ["维生素C"],
    "2026-03-16": ["维生素C", "钙片"]
  }
}
```

**数据保留：**
- 药物列表：永久保留
- 服药记录：保留 30 天

## 5. 非功能需求

### 5.1 性能
- 页面首屏加载 < 2 秒
- 交互响应 < 100ms

### 5.2 兼容性
- Chrome 80+
- Safari 14+
- Edge 80+
- 移动端响应式适配

### 5.3 隐私
- 数据仅保存在用户本地设备
- 不上传任何服务器
- 无需注册登录

### 5.4 可用性
- 界面简洁，字体清晰
- 按钮大而明显，适合老年人
- 重要操作有二次确认

## 9. 风险与限制

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 浏览器通知被屏蔽 | 提醒失效 | 引导用户开启，或使用系统级提醒 |
| LocalStorage 清除 | 数据丢失 | 提示用户定期备份 |
| 页面关闭后无法提醒 | 提醒失效 | 提示用户保持标签页打开 |
